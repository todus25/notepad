In JavaScript every expression has a value. The values of the simplest expressions are trivial: the expression 5 has the value 5, the expression "hello"+"world" has the value "hello world", an arithmetic expression such as 2+3 obviously has the value 5.

But a function call is also an expression – e.g. square(2). Its value depends on what you put next to the return keyword when you defined the function. If you have no return keyword at all, the value is undefined. In your example, the value depends on the argument, which is 2 in this case, so square(2) has the value 4 (an integer value). Now what you do with that value is up to you, just as with the basic values such as "hello". You can store it in a variable, print it using console.log(), add it to the end of some Array, pass it to some function or just ignore it (make JavaScript just forget about it):

var x = square(2);                // store in variable
console.log( square(2) );         // print to console
square( square(2) );              // pass to a function
[].push( square(2) );   // store in Array (result: [4])
square(2);              // do nothing (forget the value)

I guess you have to "return" the value, because sometimes you want to compute sth that you need for another computation. So only the 2nd value needs to be shown or returned, while the first value is only needed for another calculation. In other words, the output of a function might be an intermediary result and you are only interested in the end result, so you have to specify which result has to be returned to the user.
